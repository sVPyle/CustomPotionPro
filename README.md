# 🧪 CustomPotionsPro [1.21.X]

[![Java](https://img.shields.io/badge/Java_21-FF1111?style=flat-square&logo=Javascript&logoColor=white)]()

[![Platform](https://img.shields.io/badge/GitHub-gray?logo=github)](https://github.com/sVPyle/CustomPotionPro)

**CustomPotionsPro** — это ультимативный фреймворк для создания уникальных RPG-артефактов, магических эссенций и предметов с кастомными 3D-моделями. Используя мощь компонентов Minecraft 1.21.1, плагин позволяет переосмыслить систему предметов на вашем сервере.

---

## 🔥 Ключевые Особенности (Mega-Features)

### 💎 Продвинутая Визуализация (Mega-Models)
*   **CustomModelData**: Полная поддержка 3D-моделей из вашего ресурспака.
*   **Base64 Skins**: Создавайте артефакты на базе голов игроков, используя только Hash текстуры (не требует скина игрока).
*   **Glint Control**: Принудительное наложение эффекта зачарования (блеска) на любой предмет без отображения текста чар.
*   **Rarity System**: Использование ванильных компонентов редкости (Common, Rare, Epic) для изменения цвета рамки предмета.

### ⚡ Динамические Триггеры (Action-Events)
*   **ON_DRINK / ON_CLICK**: Автоматическое распознавание типа использования (зелье выпивается, блок активируется по ПКМ).
*   **Passive-Effects**: Выдавайте эффекты (скорость, сила, регенерация) игроку, пока он просто держит артефакт в основной руке.
*   **Infinite Items**: Создавайте многоразовые артефакты, которые не исчезают после использования (например, "Посох Жизни").

### 🌈 Современный Дизайн
*   **MiniMessage Support**: Полная поддержка HEX-цветов и градиентов в названиях и описаниях.
*   **PlaceholderAPI**: Интеграция плейсхолдеров в Lore и консольные команды.
*   **Clean UI**: Автоматическое скрытие мусорных ванильных описаний ("Нет эффектов", "Урон ближнего боя").

## 🛠 Техническая Документация

### Команды

| Команда | Описание | Право |
| :--- | :--- | :--- |
| `/cp give <player> <id>` | Выдает кастомный предмет игроку | `custompotions.admin` |
| `/cp reload` | Мгновенно обновляет конфиг и рецепты | `custompotions.admin` |

### Структура Эффектов
Формат записи эффектов в `config.yml`:
1.  **Зелье**: `ID_ЭФФЕКТА:ДЛИТЕЛЬНОСТЬ:СИЛА` (Пример: `SPEED:30:2`)
2.  **Команда**: `"/команда"` (Пример: `"/broadcast %player% использовал магию!"`)

## 🎨 Настройка 3D-моделей и Текстур (Resource Pack)

CustomPotionsPro использует стандартную механику Minecraft — **CustomModelData**. Это позволяет одному предмету (например, `IRON_INGOT`) иметь тысячи разных обликов без замены оригинальной текстуры.

### 1. Подготовка модели (Blockbench)
1. Создайте модель в **Blockbench** (формат Java Block/Item).
2. Экспортируйте файл в формате `.json`.
3. Назовите его уникально, например `soul_potion.json`.

### 2. Настройка ресурспака
В вашем ресурспаке перейдите по пути:
`assets/minecraft/models/item/`

Найдите (или создайте) файл материала, который вы указали в `config.yml` (например, `iron_ingot.json`):

```json
{
  "parent": "minecraft:item/generated",
  "textures": {
    "layer0": "minecraft:item/iron_ingot"
  },
  "overrides": [
    { "predicate": { "custom_model_data": 1005 }, "model": "item/custom/soul_potion" }
  ]
}
```
Здесь custom_model_data: 1005 — это уникальный номер, который связывает предмет с вашей моделью.
### 3. Подключение в плагине
Теперь в `config.yml` просто укажите этот номер в соответствующем поле:
```yaml
potions:
  SoulArtifact:
    Material: IRON_INGOT   # Базовый материал из ресурспака
    model-id: 1005         # Должен совпадать с custom_model_data в JSON
    Name: "<#55ffff>Артефакт Душ"
    # ... остальные настройки
```
## 💡 Продвинутые возможности моделей
В нашем плагине реализованы дополнительные подмеханики для визуала:
- **Make-Edible: true**: Если ваша модель выглядит как еда или напиток, включите этот параметр. Игрок будет видеть анимацию "поедания" модели, даже если базовый материал — железо или палка.
- **Enchanted-Glint: true**: Накладывает эффект магического сияния поверх вашей 3D-модели.
- **Skull-Hash**: Если вы используете `Material: PLAYER_HEAD`, вам не нужен ресурспак. Просто вставьте Hash текстуры, и плагин сам загрузит её из облака Mojang.
---
### Что это дает пользователю:
1. **Понимание связи**: Пользователь видит прямую связь между JSON-файлом ресурспака и конфигом плагина.
2. **Blockbench**: Упоминание популярного софта помогает новичкам сориентироваться, где брать модели.
3. **Механика**: Объяснение того, почему `IRON_INGOT` вдруг можно "есть", снимает вопросы по работе `Make-Edible`.

## 📦 Сборка Проекта
Для компиляции проекта выполните следующие шаги:
1. Убедитесь, что у вас установлена Java 21.
2. Создайте папку `libs` в корне проекта.
3. Поместите туда `PlaceholderAPI-2.12.2.jar`.
4. Запустите сборку Maven:
```bash
mvn clean package
```
5. Заберите готовый файл из папки `target/CustomPotionsPro-1.0.jar`.
